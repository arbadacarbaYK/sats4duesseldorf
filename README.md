# Satoshis für Berlin – die Millionenchallenge

**Gesamtbounty: 1.000.000 sats (nur in Bitcoin auszahlbar)**

Echte Bitcoin-Zahlungen vor Ort belegen, BTCMap aktuell halten, Sats verdienen.

**[Zur Übersicht](https://satoshiinberlin.github.io/sats4berlin/)** | **[Check einreichen](https://satoshiinberlin.github.io/sats4berlin/submit.html)** | **[Regeln](RULES.md)**

---

## Worum geht es?

BTCMap (btcmap.org) listet Orte, an denen man mit Bitcoin bezahlen können soll. Solche Informationen ändern sich schnell. Diese Challenge sorgt dafür, dass Berliner Einträge regelmäßig durch echte Käufe vor Ort überprüft und bei Bedarf aktualisiert werden.

## Schnellstart

1. **Übersicht öffnen:** [satoshiinberlin.github.io/sats4berlin](https://satoshiinberlin.github.io/sats4berlin/)
2. **Ort wählen:** Suche einen Ort mit Status „verfügbar" (grün)
3. **Vor Ort kaufen:** Bezahle in Bitcoin (Lightning oder On-Chain)
4. **Nachweise sammeln:** Bon, Zahlungsbestätigung, Foto vom Ort, öffentlicher Post
5. **Check einreichen:** [Formular ausfüllen](https://satoshiinberlin.github.io/sats4berlin/submit.html)
6. **Sats kassieren:** Nach Prüfung wird der Bounty ausgezahlt

## Wer kann mitmachen?

Jeder. Keine Vorkenntnisse nötig.
Bedingung: Du machst vor Ort einen echten Kauf und bezahlst in Bitcoin.

## Erforderliche Nachweise (alle vier)

1. **Kaufbeleg** (Bon o.ä.; sensible Daten schwärzen)
2. **Nachweis der Bitcoin-Zahlung** (Bestätigung „bezahlt", Betrag/Datum sichtbar; persönliche Daten schwärzen)
3. **Ort erkennbar** (Foto/Video von Schild, Eingang, Kasse o.ä.; keine Personen filmen)
4. **Öffentlicher Beweis-Post** mit Foto/Video und den Worten **„Berlin"** und **„Bitcoin"** (Link wird eingereicht)

## Bounty-Staffelung

### Basisbounty (nach Alter des letzten Checks)

| Alter seit letztem Check | Bounty |
|--------------------------|--------|
| 3–6 Monate | 10.000 sats |
| 6–12 Monate | 13.000 sats |
| 12–24 Monate | 17.000 sats |
| >24 Monate | 21.000 sats |

### Kritische Änderung
Für wichtige Änderungen (nimmt kein Bitcoin mehr, geschlossen, umgezogen):
- **21.000 sats** (unabhängig vom Alter)

### Neueintrag
- **21.000 sats**, Auszahlung erst nach Bestätigung durch zwei weitere Bitcoiner
- Erst wenn 3 gültige Checks vorliegen, wird das Bounty freigegeben

## Aktivitätsfaktor (bis 2,0×)

Die Auszahlung wird mit einem Aktivitätsfaktor multipliziert:

| Checks in 90 Tagen | Faktor |
|--------------------|--------|
| 0–1 | 1,0× |
| 2–4 | 1,2× |
| 5–9 | 1,5× |
| ≥10 | 2,0× |

**Auszahlung = Bounty × Aktivitätsfaktor**

## Cooldown

Ein Ort kann frühestens **3 Monate** nach dem letzten gültigen Check erneut geprüft werden.

## Auszahlung

- Bounties werden nur in **Bitcoin** ausgezahlt
- Auszahlung an **Lightning-Adresse** oder per **eCash**
- Anonymisiert – keine Klarnamen erforderlich

## Budget & Laufzeit

- Das Programm läuft, bis insgesamt **1.000.000 sats** ausgeschüttet sind
- Solange Budget verfügbar ist, kann ausgezahlt werden
- Dies ist ein Pilot: Regeln können laufend verbessert werden

## Datenquellen

- Ortsdaten: [BTCMap](https://btcmap.org) / [OpenStreetMap](https://www.openstreetmap.org)
- Daten werden zweimal täglich automatisch synchronisiert (00:00 und 12:00 Uhr)
- © OpenStreetMap contributors (ODbL)
